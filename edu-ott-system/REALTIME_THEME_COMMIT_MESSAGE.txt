feat(settings): add real-time theme sync across devices

Enable instant theme synchronization between Web and Mobile using Socket.IO.

## Problem

Theme changes took ~1 minute to sync between devices and required page reload.
Users had to manually refresh to see theme changes from other devices.

## Solution

Added Socket.IO event `settings_changed` that broadcasts theme updates to all
user's devices in real-time.

## Changes

### Backend:
- backend/controllers/settingsController.js
  * Emit `settings_changed` socket event when theme is updated
  * Event sent only to user's own devices for security

### Frontend Web:
- frontend/web/src/contexts/ThemeContext.jsx
  * Listen for `settings_changed` socket event
  * Update theme instantly without API call
  * Dynamic import to avoid circular dependency

### Frontend Mobile:
- frontend/mobile/context/auth.tsx
  * Listen for `settings_changed` socket event
  * Reload settings to sync with server
  * Auto-update theme via useColorScheme hook

## Features

✅ Theme syncs in < 150ms across all devices
✅ Works for Web → Mobile and Mobile → Web
✅ Works across multiple browser tabs
✅ Optimistic updates for instant UX
✅ Graceful handling of offline/online transitions
✅ Secure (only user's devices receive events)
