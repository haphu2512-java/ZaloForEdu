ROOT CAUSE:
- Code was checking createdBy field to determine owner
- When ownership transfers, ownerId changes but createdBy stays same
- This caused old owner to still show as owner
- Solution: Only check ownerId, ignore createdBy

LOGIC:
- ownerId = current group owner (can be transferred)
- createdBy = original creator (never changes)
- adminIds = list of admins (old owner added here after transfer)
- markAdminMessages = setting to show FaCrown/FaStar icons for admin messages
- FaCrown (gold #f59e0b) = owner messages
- FaStar (primary color) = admin messages
