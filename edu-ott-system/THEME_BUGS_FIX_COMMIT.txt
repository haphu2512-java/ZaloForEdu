fix(theme): fix backend crash and theme persistence bugs

Fix 3 critical theme-related bugs affecting all users.

## Bugs Fixed

### Bug 1: Backend crash when updating theme
- Error: "Cannot find module './socketService'"
- Cause: Incorrect require path in settingsController
- Fix: Changed `./socketService` to `../services/socketService`

### Bug 2: Theme persists after logout
- Issue: User A's theme shows for User B after logout/login
- Cause: Theme cache not cleared on logout
- Fix: Clear localStorage/AsyncStorage on logout

### Bug 3: Real-time sync not working
- Cause: Bug 1 prevented socket events from emitting
- Fix: Fixed require path enables socket events

## Changes

### Backend:
- backend/controllers/settingsController.js
  * Fixed require path: `../services/socketService`

### Frontend Web:
- frontend/web/src/store/authStore.js
  * Clear `app-theme-mode` from localStorage on logout

- frontend/web/src/contexts/ThemeContext.jsx
  * Check token before loading theme
  * Use system default when not logged in

### Frontend Mobile:
- frontend/mobile/utils/authService.ts
  * Clear `userThemeMode` from AsyncStorage on logout

## Expected Behavior

✅ Backend no longer crashes when updating theme
✅ Theme cleared on logout (each user has own theme)
✅ Real-time sync works instantly (< 1 second)
